设计模式之工厂模式

Apr 4, 2019

简单工厂

  • 描述

    这个模式本身很简单，而且使用在业务较为简单的情况下

  • 关键点

    1. 工厂类角色：这是本模式的核心，含有一定的业务逻辑和判断逻辑，在 Java 中它往往由一个类实现。

    2. 抽象产品角色：它一般是具体产品继承的父类和实现的接口，在 Java 中由接口或者抽象类实现。

    3. 具体产品角色：工厂类所创建的对象就是此角色的实例，在 Java 中由一个具体类实现。

  • 问题

    1. 违反了开闭合原则

    2. 扩展性较差，如果新增具体产品，需要修改工厂类。

  • 开闭原则

    1. 对扩展开放，对修改关闭

    2. 一个软件应该通过扩展来实现变化，而不是通过修改已有的代码来实现变化。

  • 练习实例

工厂方法

  • 描述

    定义一个创建对象的工厂接口

  • 关键点

    1. 抽象工厂角色：工厂方法的核心，必须继承的父类

    2. 具体工厂角色：创建具体产品的对象

    3. 抽象产品角色：具体产品继承的父类或者实现的接口

    4. 具体产品角色：具体工厂角色所创建的对象

  • 优点：

    1. 符合开闭原则：新增一种产品时，只需要增加相应的具体产品类和相应的工厂子类即可

    2. 符合单一职责原则：每个具体工厂类只负责创建对应的产品

  • 练习实例

抽象工厂

  • 描述：

    当我们需要对一组产品同时初始化，并且每个产品都有多种类型的时候，就需要抽象工厂模式

  • 优缺点

    优点

    1. 可以将客户从具体产品中解耦
    2. 抽象工厂可以让对象创建更被追踪
    3. 同时将对象创建与使用解耦

    缺点

    1. 抽象工厂是将一组相关的产品集合起来，如果需要扩展这组产品，就需要改变接口，意味着每个子类都需要改变接口

  • 练习实例

使用总结

工厂就是用于生产产品（也就是创建实例），当我们有多种类型的产品时需要工厂帮我们选择特定的类去实例化
